# Spec: Πολλαπλά σετ χιλιοστών, συντελεστής ορόφου ανελκυστήρα & εξαιρέσεις δαπανών

**Date:** 2026-06-21
**Status:** Approved (brainstorming)
**Σχετικά:** Επεκτείνει το [2026-06-19-building-millesimes-autocalc-design.md](2026-06-19-building-millesimes-autocalc-design.md) (το follow-up «πολλαπλά σετ» που είχε αναβληθεί). Βλ. και [KOINOXRISTA_ANALYSIS.md](../../../KOINOXRISTA_ANALYSIS.md).

## Σκοπός

Να υποστηρίξει ρεαλιστικά κοινόχρηστα όπου:
- διαφορετικοί τύποι δαπανών κατανέμονται με διαφορετική βάση (γενικά / ανελκυστήρα / θέρμανση / ισόποσα),
- ο ανελκυστήρας επιβαρύνει περισσότερο τους ψηλούς ορόφους (συντελεστής ορόφου),
- συγκεκριμένες μονάδες (συνήθως καταστήματα ισογείου) **εξαιρούνται** από συγκεκριμένες κατηγορίες δαπανών,
- ένας **μη-default κανονισμός πολυκατοικίας** μπορεί να υπερισχύει τόσο των τιμών χιλιοστών όσο και της μεθόδου κατανομής ανά κατηγορία,
- η εκτύπωση εξηγεί διαφανώς πώς προέκυψε κάθε χρέωση.

## Αρχή σχεδίασης

**Ο κανονισμός/όροφος δεν μπαίνει στη δαπάνη — μπαίνει στα χιλιοστά και στη μέθοδο κατανομής.**
Η δαπάνη φέρει μόνο `category`. Όλη η λογική (όροφος, κανονισμός, εξαιρέσεις) ζει σε προ-υπολογισμένα δεδομένα μονάδας/κτηρίου. Αλλαγή κανονισμού → καμία ιστορική δαπάνη δεν αλλάζει.

## 1. Μοντέλο δεδομένων

### 1.1 `Unit` — τρία σετ χιλιοστών + πηγή ανά σετ

```prisma
millesimes                Float?              // γενικά (υπάρχει ήδη)
millesimesElevator        Float?
millesimesHeating         Float?
millesimesSource          MillesimeSource @default(AUTO)
millesimesElevatorSource  MillesimeSource @default(AUTO)
millesimesHeatingSource   MillesimeSource @default(AUTO)

enum MillesimeSource { AUTO MANUAL }
```

- `floor` υπάρχει ήδη — τροφοδοτεί τον συντελεστή ανελκυστήρα.
- `*Source = MANUAL` σημαίνει «τιμή κανονισμού/χειροκίνητη» → ο αυτόματος επανυπολογισμός **δεν** την αγγίζει (per-cell auto-lock). «Επαναφορά σε αυτόματο» επαναφέρει σε `AUTO` και ξαναϋπολογίζει.

### 1.2 `Building` — παράμετροι ανελκυστήρα

```prisma
elevatorSurchargePerFloor  Float   @default(0.10)   // 10% ανά όροφο, ρυθμιζόμενο
elevatorExemptGroundFloor  Boolean @default(true)
```

### 1.3 `ExpenseCategory` — default μέθοδος κατά νόμο

```prisma
defaultBasis  DistributionBasis @default(GENERAL_MILLESIMES)
```

### 1.4 `BuildingCategoryOverride` — μέθοδος κατά κανονισμό (επέκταση υπάρχοντος)

```prisma
distributionBasis  DistributionBasis?   // null = κράτα το defaultBasis της κατηγορίας
```
Το υπάρχον `@@unique([buildingId, categoryId])` παραμένει το σωστό κλειδί.

### 1.5 `UnitCategoryExclusion` — matrix εξαιρέσεων (νέο)

```prisma
model UnitCategoryExclusion {
  id         String   @id @default(cuid())
  unitId     String
  unit       Unit     @relation(fields: [unitId], references: [id], onDelete: Cascade)
  categoryId String
  category   ExpenseCategory @relation(fields: [categoryId], references: [id], onDelete: Cascade)
  createdAt  DateTime @default(now())
  @@unique([unitId, categoryId])
  @@index([unitId])
  @@index([categoryId])
}
```
- **Sparse:** αποθηκεύονται ΜΟΝΟ οι εξαιρέσεις. Καμία γραμμή = συμμετέχει. Default «όλοι πληρώνουν τα πάντα» χωρίς καμία εγγραφή.
- Μόνιμος κανόνας στο διηνεκές — **όχι** per-expense override (εκτός scope, βλ. §6).

### 1.6 Enum μεθόδων κατανομής

```prisma
enum DistributionBasis {
  GENERAL_MILLESIMES
  ELEVATOR_MILLESIMES
  HEATING_MILLESIMES
  EQUAL_PER_UNIT
  METERED_70_30
}
```

## 2. Υπολογισμός χιλιοστών (καθαρές συναρτήσεις, `lib/millesimes.ts`)

Επεκτείνει την υπάρχουσα `computeMillesimes` ώστε να καλείται ανά σετ με διαφορετική βάρους/εξαιρέσεις.

**Γενικά:** `βάρος_i = areaSqm_i`.

**Ανελκυστήρα:**
```
βάρος_i = areaSqm_i × (1 + surchargePerFloor × floor_i)
αν elevatorExemptGroundFloor και floor_i == 0 → βάρος_i = 0
```

**Θέρμανση:** `βάρος_i = όγκος_i` (ή `areaSqm_i` αν δεν υπάρχει ύψος) — ογκομετρικά.

Σε κάθε σετ: `χιλιοστά_i = round(βάρος_i / Σβάρη × 1000, 2)`, με διόρθωση υπολοίπου στη μεγαλύτερη μονάδα ώστε άθροισμα = ακριβώς 1000,00 (ίδιος κανόνας με το υπάρχον spec). Μονάδες με βάρος 0 ή χωρίς δεδομένα → `null`/0, εκτός κατανομής.

Ο επανυπολογισμός ενημερώνει **μόνο** κελιά με `Source = AUTO`.

## 3. Αλγόριθμος κατανομής δαπάνης (έκδοση κοινοχρήστων)

Για κάθε `expense` του μήνα:

1. **Μέθοδος:** `basis = buildingOverride(categoryId)?.distributionBasis ?? category.defaultBasis`.
2. **Συμμετέχοντες:** όλες οι μονάδες ΕΚΤΟΣ όσων έχουν `UnitCategoryExclusion(unitId, categoryId)`.
3. **Βάρη ανά μέθοδο:**
   - `GENERAL/ELEVATOR/HEATING_MILLESIMES` → το αντίστοιχο σετ χιλιοστών της μονάδας,
   - `EQUAL_PER_UNIT` → ίσο βάρος ανά συμμετέχουσα μονάδα,
   - `METERED_70_30` → βλ. βήμα 5.
4. **Ανακατανομή στο 1000:** τα βάρη των συμμετεχόντων κανονικοποιούνται ώστε άθροισμα = 1000 (το μερίδιο των εξαιρεμένων απορροφάται αναλογικά). `μερίδιο_i = amount × βάρος_i / Σβάρη_συμμετεχόντων`.
5. **METERED_70_30:** `μερίδιο_i = amount × [0.70 × (μέτρηση_i / Σμετρήσεις) + 0.30 × (χιλ.θέρμ_i / 1000_συμμετεχόντων)]`. Χωρίς μετρήσεις → fallback σε καθαρά χιλιοστά θέρμανσης.

Η δαπάνη παραμένει «χαζή»: φέρει μόνο `categoryId`. Το αποτέλεσμα γράφεται στα υπάρχοντα `ExpenseAllocation`.

## 4. UI — παραμετροποίηση (3 σημεία στις παραμέτρους κτηρίου)

Σχεδιαστική αρχή: **μηδενική παραμετροποίηση by default**, ευελιξία όταν χρειάζεται, ένα εργαλείο να μάθει ο χρήστης.

### 4.1 Πίνακας χιλιοστών
- Όλες οι μονάδες × 3 σετ (Γενικά / Ανελκ. / Θέρμ.) σε **έναν** πίνακα.
- Header: πεδίο «επιβάρυνση ανελκυστήρα/όροφο» (default 10%) + checkbox «εξαίρεση ισογείου».
- **Inline edit + αυτο-κλείδωμα:** γράφεις σε κελί → γίνεται `MANUAL`, εμφανίζεται 🔒 και κίτρινη ένδειξη (διαφέρει από αυτόματο).
- **Ζωντανά σύνολα** ανά σετ με ✓ όταν = 1000,00 (κόκκινο/✗ αλλιώς).
- Κουμπιά: «Επανυπολογισμός αυτομάτων» (αγγίζει μόνο AUTO κελιά), «Επαναφορά κλειδωμένων σε αυτόματο» (ανά κελί / ανά κτήριο).
- Η προεπισκόπηση υπολογίζεται client-side με την ίδια `lib/millesimes.ts` (κοινός κώδικας με το server action).

### 4.2 Tab «Κατανομή»
- Πίνακας: κατηγορία | default (νόμος, read-only) | μέθοδος κτηρίου (dropdown) | ένδειξη.
- Dropdown: Γενικά χιλιοστά · Χιλιοστά ανελκυστήρα · Χιλιοστά θέρμανσης · Ισόποσα/μονάδα · 70/30 μετρητής.
- Κίτρινη ένδειξη «κανονισμός» όπου `distributionBasis != defaultBasis`. «Επαναφορά σε default» ανά γραμμή (διαγράφει το override).

### 4.3 Matrix συμμετοχής (εξαιρέσεις)
- Checkboxes μονάδα × κατηγορία (☑ = πληρώνει). Default όλα ☑ (καμία εγγραφή).
- Ξετσεκάρισμα → δημιουργεί `UnitCategoryExclusion`. Bulk action: «εξαίρεσε όλα τα καταστήματα από <κατηγορία>».

## 5. Εκτύπωση — διαφανής ανάλυση

Κάθε ειδοποίηση μονάδας δείχνει **ανά δαπάνη**:
- τίτλο & μερίδιο σε €,
- συνολική δαπάνη + μέθοδος κατανομής (σε λέξεις),
- συμμετέχοντες «N/M» + ονομαστικά οι εξαιρεμένοι,
- χιλιοστά μονάδας (με σημείωση «αναπροσαρμοσμένα» όταν έγινε re-normalize),
- τον αριθμητικό υπολογισμό που βγάζει το ποσό (για METERED, ανάλυση 70/30),
- **Σύνολο μήνα**.

Στόχος: ο ιδιοκτήτης/ένοικος να βλέπει *γιατί* πληρώνει κάθε ποσό.

---

## 6. Εκτός scope (τώρα)

- **Per-expense override** εξαιρέσεων/μεθόδου για μεμονωμένο τιμολόγιο — οι κανόνες είναι μόνιμοι (κατηγορία-επίπεδο), στο διηνεκές.
- Ελεύθερος κατάλογος σετ χιλιοστών (`MillesimeSet`) — μένουμε στα 3 σταθερά σετ.
- Μη-γραμμικός πίνακας συντελεστή ορόφου ανά όροφο — μόνο ενιαίο % ανά κτήριο.
- Διπλοί πίνακες χιλιοστών residential/commercial — καλύπτεται επαρκώς από εξαιρέσεις + μέθοδο.

## 7. Δοκιμές

Unit tests (`lib/millesimes.ts`, καθαρές):
1. Σετ ανελκυστήρα: ισόγειο εξαιρείται (βάρος 0), ψηλότεροι όροφοι αναλογικά βαρύτεροι, άθροισμα 1000,00.
2. Επανυπολογισμός σέβεται `MANUAL` κελιά (δεν τα αλλάζει), ενημερώνει μόνο `AUTO`.
3. Κατανομή με εξαίρεση: μονάδα εξαιρεμένη → 0 €, υπόλοιποι re-normalize, άθροισμα μεριδίων = συνολική δαπάνη (σεντ-ακριβές).
4. `EQUAL_PER_UNIT`: ίσα μερίδια μεταξύ συμμετεχόντων.
5. `METERED_70_30`: σωστός συνδυασμός· χωρίς μετρήσεις → fallback σε χιλιοστά θέρμανσης.
6. Override κανονισμού: `distributionBasis` υπερισχύει του `defaultBasis`.
