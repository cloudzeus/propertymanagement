# Spec: Αυτόματος υπολογισμός χιλιοστών κτηρίου

**Date:** 2026-06-19
**Status:** Approved (brainstorming)

## Σκοπός

Δυνατότητα αυτόματου υπολογισμού των χιλιοστών (‰) όλων των μονάδων ενός κτηρίου,
κατανεμημένων αναλογικά στα τετραγωνικά (εξ αδιαιρέτου αναλογία κατά νόμο, όταν δεν
υπάρχει κανονισμός). Ενεργοποιείται μέσω νέας ενέργειας στο kebab-dropdown κάθε κτηρίου,
με modal προεπισκόπησης πριν την εφαρμογή.

## Πεδίο εφαρμογής (scope)

- Ένα πεδίο `Unit.millesimes` (όχι τα 3 σετ general/elevator/heating — εκτός scope τώρα).
- Επίπεδο κτηρίου: ο υπολογισμός αφορά όλες τις μονάδες ενός κτηρίου μαζί.

## Κανόνας υπολογισμού

Για κάθε μονάδα `i` με `areaSqm` συμπληρωμένο:

```
millesimes_i = round( areaSqm_i / Σ(areaSqm μονάδων με τ.μ.) × 1000 , 2 )
```

- **Μορφή:** δεκαδικά, 2 ψηφία.
- **Διόρθωση υπολοίπου:** μετά τη στρογγυλοποίηση, η διαφορά του αθροίσματος από το
  1000,00 (π.χ. ±0,01–0,02) προστίθεται/αφαιρείται από τη μονάδα με το μεγαλύτερο
  εμβαδό, ώστε το τελικό άθροισμα να είναι **ακριβώς 1000,00**.
- **Μονάδες χωρίς τ.μ.:** εξαιρούνται από την κατανομή και τα χιλιοστά τους
  **μηδενίζονται** (`millesimes → null`), ώστε το άθροισμα του κτηρίου να παραμένει πάντα
  ακριβώς 1000. Εμφανίζονται με προειδοποίηση στο modal.
- **Καμία μονάδα με τ.μ.:** δεν γίνεται υπολογισμός· μήνυμα σφάλματος, απενεργοποιημένο
  κουμπί εφαρμογής.

## Αρχιτεκτονική

### Καθαρή συνάρτηση — `lib/millesimes.ts`

```ts
export type MillesimeInput = { id: string; areaSqm: number | null };
export type MillesimeResult = { id: string; millesimes: number | null }; // null = χωρίς τ.μ., αμετάβλητο

export function computeMillesimes(units: MillesimeInput[]): MillesimeResult[];
```

- Χωρίς εξαρτήσεις (καθαρή), δοκιμάζεται απομονωμένα.
- Επιστρέφει νέα τιμή ‰ για μονάδες με τ.μ.· `null` για μονάδες χωρίς τ.μ.

### Server action — `app/actions/buildings.ts`

```ts
export async function recalculateMillesimes(buildingId: string):
  Promise<{ updated: number } | { error: string }>
```

- `requireStaff`-style auth (SUPER_ADMIN/ADMIN/MANAGER/PROPERTY_ADMIN, όπως τα υπόλοιπα building actions).
- Διαβάζει `units` του κτηρίου (`id`, `areaSqm`).
- Καλεί `computeMillesimes`, ενημερώνει σε `prisma.$transaction` (updateMany ανά τιμή ή
  σειριακά updates) μόνο τις μονάδες με τ.μ.
- `revalidatePath` της σελίδας ιδιοκτησίας.

### UI — modal προεπισκόπησης

Νέα ενέργεια στο building dropdown ([CustomerTree.tsx](../../../app/(dashboard)/super-admin/customers/CustomerTree.tsx) γραμμή ~109),
**πάνω από τη «Διαγραφή»**:

```
{ label: "Υπολογισμός χιλιοστών", icon: <RiCalculatorLine />, onClick: () => setModal({ kind: "millesimes", building: b }) }
```

Modal `MillesimesModal`:
- Τίτλος: «Υπολογισμός χιλιοστών — {όνομα κτηρίου}».
- Πίνακας: Μονάδα | τ.μ. | Παλιά ‰ | Νέα ‰ (χρωματική ένδειξη όταν αλλάζει).
- Footer: Σύνολο τ.μ. · Σύνολο νέων ‰ (=1000,00).
- Προειδοποίηση (κίτρινη) αν υπάρχουν μονάδες χωρίς τ.μ.: «N μονάδες χωρίς τ.μ. — δεν θα ενημερωθούν».
- Σφάλμα + disabled κουμπί αν καμία μονάδα δεν έχει τ.μ.
- Κουμπιά: «Ακύρωση» / «Εφαρμογή σε όλες» → `recalculateMillesimes` → `router.refresh()`.

Ο υπολογισμός για προεπισκόπηση γίνεται client-side καλώντας την ίδια `computeMillesimes`
(κοινός κώδικας), ώστε προεπισκόπηση και αποθήκευση να συμφωνούν.

## Δοκιμές

Unit tests για `computeMillesimes`:
1. Ισομερή τ.μ. (3 μονάδες × 100 τ.μ.) → 333,34 / 333,33 / 333,33 (άθροισμα 1000,00).
2. Άνιση κατανομή με στρογγυλοποίηση → άθροισμα ακριβώς 1000,00, υπόλοιπο στη μεγαλύτερη.
3. Μονάδες χωρίς τ.μ. → επιστρέφονται με `null`, δεν συμμετέχουν στην κατανομή.
4. Καμία μονάδα με τ.μ. → όλες `null` (το action επιστρέφει σφάλμα πριν φτάσει εδώ).
5. Μία μονάδα με τ.μ. → 1000,00.

## Εκτός scope (τώρα)

- Χειροκίνητη υπέρβαση ανά μονάδα πέραν της υπάρχουσας επεξεργασίας στο UnitModal.
- Υπολογισμός βάσει κανονισμού πολυκατοικίας.

## Προγραμματισμένο follow-up (επόμενο βήμα)

**Πολλαπλά σετ χιλιοστών** (γενικά / ανελκυστήρα / θέρμανσης κ.λπ.). Σχεδιαστική
κατεύθυνση ώστε ο τωρινός κώδικας να μην εμποδίζει την επέκταση:
- Η `computeMillesimes` παραμένει γενική (κατανομή ποσότητας × 1000) ώστε να καλείται
  ξεχωριστά ανά σετ με διαφορετική βάση/εξαιρέσεις (π.χ. ισόγειο εκτός ανελκυστήρα).
- Το μοντέλο πιθανότατα χρειάζεται είτε πρόσθετα πεδία στο `Unit` (`millesimesElevator`,
  `millesimesHeating`, …) είτε πίνακα `MillesimeSet`. Θα σχεδιαστεί σε ξεχωριστό spec.
